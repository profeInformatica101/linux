Comandos básicos del sistema de archivos

Navegación, gestión de ficheros y directorios. Base de cualquier examen de administración Linux.

ls Lista el contenido de un directorio Examen
-l formato largo · -a incluye ocultos · -h tamaños legibles · -t ordena por fecha
bash — terminal 
# Listar con detalle todos los ficheros (incluidos ocultos)
ls -lah /var/log

cp mv rm Copiar, mover/renombrar y eliminar ficheros Precaución
rm -rf elimina sin confirmación y sin papelera. Usar con mucha precaución.
bash 
# Copiar directorio completo (-r recursivo, -p preserva permisos)
cp -rp /var/www/html /var/www/html_backup

# Mover (también sirve para renombrar)
mv informe_v1.txt informe_final.txt

# Mover múltiples ficheros a un directorio
mv *.log /var/log/archivados/

# Eliminar directorio con todo su contenido
rm -rf /tmp/pruebas/

# Eliminar con confirmación interactiva (más seguro)
rm -ri /tmp/pruebas/

Permisos y usuarios

Sistema de permisos rwx de Unix/Linux. Uno de los temas más preguntados en oposiciones de Informática.

Estructura de permisos Desglose visual del bloque de permisos Muy preguntado
-tipo
|
rwxpropietario
|
r-xgrupo
|
r--otros
SímboloSignificadoValor octalPara ficherosPara directorios
rRead (lectura)4Leer contenidoListar con ls
wWrite (escritura)2Modificar/borrarCrear/borrar ficheros
xExecute (ejecución)1Ejecutar programaEntrar con cd
-Sin permiso0Sin acceso
Ejemplo: rwxr-xr-- = propietario 7 (4+2+1), grupo 5 (4+0+1), otros 4 (4+0+0) → octal 754
chmod Cambiar permisos de ficheros y directorios Examen
bash 
# Modo octal: rwxr-xr-- (propietario=7, grupo=5, otros=4)
chmod 754 script.sh

# Modo simbólico: añadir ejecución al propietario
chmod u+x script.sh

# Quitar escritura a grupo y otros
chmod go-w fichero_config.conf

# Aplicar permisos recursivamente a todo un directorio
chmod -R 755 /var/www/html/

# Caso típico examen: script solo ejecutable por el propietario
chmod 700 backup.sh
chown Cambiar propietario y grupo de un fichero
Sintaxis: chown usuario:grupo fichero — requiere privilegios de root (usar con sudo)
bash 
# Cambiar propietario y grupo
chown www-data:www-data /var/www/html/index.php

# Solo cambiar el propietario (sin tocar el grupo)
chown alumno /home/alumno/notas.txt

# Solo cambiar el grupo
chown :desarrolladores proyecto.sh

# Recursivo: asignar todo el directorio web al usuario nginx
sudo chown -R nginx:nginx /var/www/miapp/

Variables y entrada/salida

Declaración de variables, lectura de datos del usuario y volcado de texto. Fundamento del scripting.

Variables Declaración, uso y variables especiales Examen
Sin espacios alrededor del =. Para usar una variable, precederla con $ o ${}.
bash 
# Declarar variables (sin espacios alrededor del =)
NOMBRE="María García"
EDAD=23
DIRECTORIO="/home/alumno"

# Usar variables
echo "Hola, $NOMBRE, tienes ${EDAD} años"
Hola, María García, tienes 23 años

# Variables especiales del sistema
echo "Script: $0"    # nombre del script
echo "Arg 1: $1"    # primer argumento
echo "Total args: $#" # número de argumentos
echo "PID: $$"       # PID del proceso actual
echo "Último código: $?" # código de retorno anterior
read echo Leer entrada del usuario / mostrar salida
bash — script interactivo 
#!/bin/bash
# Script de registro de usuario

echo -n "Introduce tu nombre de usuario: "
read USUARIO

read -p "Introduce tu edad: " EDAD
read -sp "Introduce contraseña: " PASS  # -s = ocultar entrada

echo  # salto de línea tras contraseña oculta
echo "Usuario '$USUARIO' con edad $EDAD registrado."

# Aritmética con variables numéricas
SUMA=$(( $EDAD + 10 ))
echo "En 10 años tendrás: $SUMA"

if / elif / else Estructura condicional completa Tipo examen
bash — ejercicio tipo oposición 
#!/bin/bash
# Ejercicio: verificar si un fichero de log existe y tiene contenido

FICHERO="/var/log/syslog"

if [[ -f "$FICHERO" && -r "$FICHERO" ]]; then
    LINEAS=$(wc -l < "$FICHERO")
    if [[ $LINEAS -gt 1000 ]]; then
        echo "AVISO: $FICHERO tiene $LINEAS líneas. Considerar rotación."
    elif [[ $LINEAS -gt 0 ]]; then
        echo "OK: Fichero con $LINEAS líneas."
    else
        echo "VACÍO: El fichero no tiene contenido."
    fi
else
    echo "ERROR: $FICHERO no existe o no es legible."
    exit 1
fi
case Alternativa limpia a múltiples if-elif

    S|D)        echo "Fin de semana" ;;
    *)          echo "Entrada no válida" ; exit 1 ;;
esac

Bucles

Iteración sobre listas, rangos y condiciones. Herramienta clave para automatizar tareas repetitivas.

for Iterar sobre una lista o rango de valores Examen
bash — recorrido de ficheros 
#!/bin/bash
# Procesar todos los ficheros .csv de un directorio

for FICHERO in /datos/*.csv; do
    if [[ -f "$FICHERO" ]]; then
        LINEAS=$(wc -l < "$FICHERO")
        echo "$FICHERO → $LINEAS líneas"
    fi
done

# Iterar sobre un rango numérico (1 al 5)
for i in {1..5}; do
    echo "Creando backup_$i.tar.gz"
done

# Sintaxis estilo C (con paso)
for (( i=0; i<10; i+=2 )); do
    echo "Número par: $i"
done
while Repetir mientras se cumpla una condición
bash — monitorización de disco 
#!/bin/bash
# Esperar a que el uso de disco baje del 90%

USO=$(df / | awk 'NR==2 {print $5}' | tr -d '%')

while [[ $USO -ge 90 ]]; do
    echo "ALERTA: Disco al $USO%. Esperando..."
    sleep 60
    USO=$(df / | awk 'NR==2 {print $5}' | tr -d '%')
done

echo "OK: Uso de disco: $USO%"

# Leer líneas de un fichero con while
while IFS= read -r LINEA; do
    echo "Procesando: $LINEA"
done < /etc/hosts

Creación y ejecución de scripts

Estructura básica, shebang, permisos de ejecución y buenas prácticas para scripts robustos.

Estructura de un script Plantilla completa con buenas prácticas Fundamental
El shebang #!/bin/bash en la primera línea indica qué intérprete debe ejecutar el script.
bash — plantilla profesional 
#!/bin/bash
# ============================================================
# Script:  backup_logs.sh
# Autor:   Administrador del sistema
# Fecha:   2024-05-03
# Desc:    Comprime y archiva logs más antiguos de 7 días
# ============================================================

set -euo pipefail  # salir si hay error / variable no definida

# Constantes
readonly LOG_DIR="/var/log"
readonly BACKUP_DIR="/backup/logs"
readonly DIAS=7

# Función de uso
uso() {
    echo "Uso: $0 [-d directorio] [-n días]"
    exit 1
}

# Verificar que el directorio de backup existe
if [[ ! -d "$BACKUP_DIR" ]]; then
    mkdir -p "$BACKUP_DIR"
fi

# Comprimir logs antiguos
find "$LOG_DIR" -name "*.log" -mtime +$DIAS -exec gzip {} \;
echo "Backup completado: $(date '+%Y-%m-%d %H:%M')"
Permisos y ejecución Hacer ejecutable y lanzar un script
bash — terminal 
# Paso 1: crear el script
nano backup_logs.sh

# Paso 2: hacer ejecutable
chmod +x backup_logs.sh
# o solo para el propietario:
chmod 700 backup_logs.sh

# Paso 3: ejecutar
./backup_logs.sh               # desde el directorio actual
bash backup_logs.sh            # sin necesitar +x
/ruta/completa/backup_logs.sh  # ruta absoluta

# Ejecutar como root
sudo ./backup_logs.sh

# Verificar sintaxis sin ejecutar
bash -n backup_logs.sh
backup_logs.sh: sin errores de sintaxis

Utilidades avanzadas

grep, awk, sed y pipes para procesamiento de texto. Potencia máxima de la línea de comandos Unix.

grep Buscar patrones en texto Examen
-i sin distinción may/min · -n número de línea · -r recursivo · -v invertir (excluir) · -E expresiones regulares extendidas
bash 
# Buscar errores en el log del sistema
grep -in "error" /var/log/syslog

# Buscar IPs con expresiones regulares
grep -E "[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}" access.log

# Contar coincidencias
grep -c "Failed password" /var/log/auth.log
47

# Buscar recursivamente en todos los .conf
grep -rl "Listen 80" /etc/apache2/
awk Procesamiento de campos y columnas de texto Avanzado
$1, $2… representan los campos de cada línea. NR = número de línea. NF = número de campos.
bash 
# Mostrar solo la columna de uso de disco (%)
df -h | awk 'NR>1 {print $5, $6}'
43% /
12% /boot
78% /home

# Sumar tamaño de todos los ficheros .log
ls -l /var/log/*.log | awk '{suma += $5} END {print "Total:", suma, "bytes"}'

# Filtrar líneas donde el campo 3 (usuario) sea "root"
awk -F':' '$3 == "root" {print $1, $6}' /etc/passwd
sed Edición de texto con sustituciones y filtros Avanzado
bash 
# Sustituir primera ocurrencia de cada línea
sed 's/http:/https:/g' config.txt

# Modificar el fichero directamente (-i hace backup con .bak)
sed -i.bak 's/puerto=80/puerto=8080/g' servidor.conf

# Eliminar líneas en blanco
sed '/^$/d' fichero.txt

# Mostrar solo líneas 10 a 20
sed -n '10,20p' /var/log/syslog
Pipes | Encadenar comandos: la salida de uno es la entrada del siguiente Muy preguntado
Los pipes (|) conectan comandos en cadena sin necesitar ficheros intermedios.
bash — pipeline realista tipo oposición 
# ¿Cuántos intentos fallidos de login hay por usuario?
# auth.log → filtrar "Failed" → extraer usuario → ordenar → contar
grep "Failed password" /var/log/auth.log \
    | awk '{print $9}' \
    | sort \
    | uniq -c \
    | sort -rn \
    | head -10

     47 root
     23 admin
     12 ubuntu
      8 oracle

# Los 5 procesos que más CPU consumen
ps aux | sort -k3 -rn | head -6

# Contar extensiones únicas en un directorio
ls /var/www/html | grep -o '\.[^.]*$' | sort | uniq -c
Ejercicio integrador — Nivel oposición

Script completo que integra comandos básicos, permisos, bucles, condicionales y pipes.

monitor_sistema.sh — ejercicio completo 
#!/bin/bash
# monitor_sistema.sh — Informe de estado del sistema

INFORME="/tmp/informe_$(date +%Y%m%d).txt"

{
  echo "=== INFORME: $(date) ==="
  echo

  # Uso de disco por partición
  echo "--- USO DE DISCO ---"
  df -h | awk 'NR==1 || $5+0 >= 70 {print $0}'
  echo

  # Usuarios conectados
  echo "--- USUARIOS ACTIVOS ---"
  who | awk '{print $1, $3, $4}'
  echo

  # Alertas de seguridad
  echo "--- FALLOS DE LOGIN (últimas 24h) ---"
  grep "Failed password" /var/log/auth.log \
      | awk '{print $9}' | sort | uniq -c | sort -rn | head -5

} > "$INFORME"

# Asignar permisos seguros al informe
chmod 600 "$INFORME"
echo "Informe generado: $INFORME"
